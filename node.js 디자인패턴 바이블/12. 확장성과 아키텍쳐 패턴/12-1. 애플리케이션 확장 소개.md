# Node.js 애플리케이션 확장

- 노드의 경우 단일스레드 특성상 여러 프로세스와 시스템에 걸쳐서 확장이 필요함

<br/>

# 확장성 3차원

### X(복제)

- 대부분 시간 비용이 저렴하고 비용 측면에서 매우 효과적
- 동일한 앱을 N번 복제하고, 각 인스턴스에서 1/N의 부하를 처리할 수 있음

<br/>

### Y(서비스/기능별 분해)

- 고유한 코드기반을 사용하는 앱을 만ㄴ드는것을 의미
- 흔히 마이크로 서비스 아키텍쳐라고 불림

<br/>

### Z(데이터 파티션 분할)

- 주로 디비에서 사용하는 개념으로 수평/수직 분할이라고 부름
