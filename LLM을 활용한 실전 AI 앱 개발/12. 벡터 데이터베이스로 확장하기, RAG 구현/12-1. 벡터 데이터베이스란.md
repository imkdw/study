# 벡터 데이터베이스란?
- 벡터 임베딩을 키로 사용하는 데이터베이스를 뜻함
- 텍스트, 이미지, 음성 등 비정형 데이터는 물론이고 서비스를 사용하는 사용자도 임베딩 모델을 통해서 벡터로 표현이 가능함
- 데이터를 적절한 임베딩 모델을 통해서 임베딩 벡터로 변환했다면 벡터 디비에 저장하고 거리게산을 통해서 유사하거나 관련이 깊은 데이터 검색이 가능함
- 즉 벡터 사이의 거리 게산에 특화된 데이터베이스임

<br>

# 딥러닝과 벡터 데이터베이스
- 딥러닝과 머신러닝의 차이점은 사람이 직접 데이터의 특징을 정의하는지 여부다
- 머신러닝은 직접 특징에 대해서 정의해야하지만, 딥러닝은 그 과정조차도 학습한다
- 딥러닝에서는 데이터만 충분하다면 모델이 알아서 특징을 뽑는 과정도 학습하는데 이를 `표현 학습`이라고 한다

<br>

### 벡터 디비를 활용하기 위한 단계
- 저장 : 저장할 데이터를 임베딩 모델을 거쳐서 벡터로 변환 후 벡터 디비에 저장
- 검색 : 검색할 데이터를 임베딩 모델을 거쳐서 벡터로 변환하고 벡터 디비에서 검색
  - 거리 계산시 유클리드 거리, 코사인 유사도, 점곱 내적을 가장 많이 사용
- 결과 반환 : 벡터 디비에서는 검색 쿼리의 임베딩과 거리가 가까운 벡터를 찾아 반환

<br>

### 벡터 디비 활용처
- 핀터레스트처럼 이미지 검색과 같이 비정형 데이터의 유사성을 기반으로 검색을 제공하는 서비스
- 사용자가 구매한 상품과 유사한 상품을 추천하는 추천 시스템 구현

<br>

# 벡터 디비 지형 파악하기

- 메타 : Faiss
- 스포티파이 : Annoy

<br>

- Chroma

- Vespa

<br>

### 벡터 기능이 추가된 디비
- ElasticSearch
- PostgreSQL
- MongoDB
- Neo4j

<br>

### 어떤 기술을 사용할지
- 벡터 저장과 검색 기능을 구현하거나 벡터 디비에가 제공하는 부가기능이 필요하지 않다면 벡터 라이브러리로 충분함
- 벡터 디비에 대한 이해도 있고 직접 오픈소스 서비스를 활용해 시스템을 구축하는걸 선호한다면 오픈소스 디비 사용
- 이해도가 부족하다면 제품 형태로 제공하는 디비 사용
- 벡터 검색 기능을 무겁게 하지 않고 기존에 구축된 디비가 있다면 벡터 기능이 추가된 디비 사용하기
